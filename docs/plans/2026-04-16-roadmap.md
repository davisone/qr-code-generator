# QRaft — Roadmap d'amélioration complète

**Date :** 2026-04-16
**Vision :** transformer QRaft d'un MVP solide en **la référence européenne du QR code dynamique avec analytics avancés**, positionnement éditorial/design fort, monétisation mûre.

**Philosophie :** moderne, différenciant, pas "AI slop". Chaque feature doit être soit **un wow-effect visuel**, soit **un vrai avantage compétitif mesurable**.

---

## Phase 0 — Fondations techniques critiques

**Objectif :** rendre le projet fiable, maintenable, scalable avant d'ajouter des features.

### 0.1 Tests automatisés (Vitest + Playwright)
- Vitest pour unitaires : `isPro()`, `parseUserAgent()`, rate-limit, expiration QR
- Playwright pour e2e : flow signup → create QR → share → scan → analytics
- Tests webhook Stripe avec mocks (3 événements critiques)
- Objectif : **60% de couverture** sur `src/lib/` et API routes

### 0.2 Monitoring production
- Sentry (tier gratuit 5k events/mois) : capture erreurs client + serveur
- PostHog self-hosted ou cloud free : product analytics (funnel signup → pro)
- UptimeRobot : ping `/api/health` toutes les 5 min
- Dashboard Vercel Analytics : Core Web Vitals en continu

### 0.3 CI/CD GitHub Actions
- Workflow `ci.yml` : lint + typecheck + tests sur chaque PR
- Workflow `deploy-preview.yml` : Vercel preview auto
- Workflow `db-check.yml` : vérifie `prisma migrate status` avant merge main
- Badge README : coverage, build, version

### 0.4 Refactor dashboard (876 → ~150 lignes)
- Découpage en composants :
  - `DashboardHeader` (stats + user menu)
  - `DashboardTabs` (QRCodes / Analytics / Map)
  - `QRCodeFilters` (dropdown type + favoris toggle + recherche)
  - `QRCodeGrid` avec `QRCodeCard` réutilisable
  - `DeleteModal` isolé
- Hooks custom :
  - `useQRCodes()` — fetch + cache avec SWR ou TanStack Query
  - `useProStatus()` — status Stripe centralisé
  - `useQRFilters()` — gestion filtres + query params URL

### 0.5 Infrastructure
- Rate limit avec **Upstash Redis** (au lieu de in-memory) → survit aux cold starts
- Cache Redis pour `geolocateIp` (TTL 24h, évite requêtes doublonnées)
- Migration webhook Stripe en **Edge Runtime** pour latence <50ms
- Remplacer ip-api.com par **MaxMind GeoLite2** self-hosted (pas de SLA sinon)

## Phase 2 — Features killer

**Objectif :** 5 fonctionnalités différenciantes qui font passer Free → Pro.

### 2.1 QR codes dynamiques (THE killer feature)
**Principe :** le QR pointe vers une URL courte QRaft (`/r/token`), mais le contenu réel est modifiable **à tout moment** sans réimprimer le QR.

- Dashboard : bouton "Modifier le contenu" sur chaque QR Pro
- Historique des modifications (versions avec timestamps)
- Option "Programmer un changement" (nouveau contenu à date X)
- Option "Redirection conditionnelle" par device/pays/horaire

### 2.2 QR templates marketplace
Bibliothèque de **50+ templates pré-designés** par cas d'usage :
- **Restaurant** : menu QR avec photo plat + allergènes
- **Événement** : QR billet avec infos + calendrier
- **Carte de visite** : vCard stylée avec photo + réseaux
- **WiFi** : carte WiFi printable avec design
- **Avis Google** : QR avec arrow "Laissez un avis"
- **Instagram** : QR avec handle + bio link
- Chaque template : preview + "Utiliser ce template"
- Pro : accès à tous · Free : 5 templates

### 2.3 Création en masse (CSV bulk)
- Upload CSV avec colonnes (name, content, type, color, logo_url)
- Preview des QR avant génération
- Export ZIP avec tous les QR + PDF "fiche par QR"
- Limite Free : 5 · Pro : 500 · Business : illimité

### 2.4 Pages de destination courtes (Bio links)
À la Linktree mais pour QR :
- Chaque QR peut pointer vers une page QRaft avec :
  - Header avec photo/logo
  - Liens vers réseaux / site / contact
  - Boutons CTA (WhatsApp, appel, itinéraire)
  - Analytics par clic de lien
- Éditeur drag-drop pour construire la page
- Templates : "Restaurant", "Créateur", "Commerce local"

### 2.5 Analytics temps réel avec heatmap
- **Live view** : voir les scans arriver en temps réel (SSE ou WebSocket)
- **Heatmap géographique** : carte mondiale avec intensité par pays
- **Conversion funnel** : scan → clic CTA → action complétée
- **Export PDF rapport** brandable (logo client, couleurs)
- **Comparaison périodes** : mois vs mois précédent
- **Alerts** : "Scan anormal", "Pic de trafic détecté"

## Phase 3 — Features business & collaboration

**Objectif :** monter en gamme vers offre Business (29€/mois).

### 3.1 Workspaces d'équipe
- Invitation membres par email
- Rôles : Owner / Admin / Editor / Viewer
- QR partagés au niveau workspace
- Activity log (qui a modifié quoi)
- Facturation centralisée

### 3.2 API publique
- Endpoints REST :
  - `POST /api/v1/qrcodes` (create)
  - `GET /api/v1/qrcodes/:id/analytics`
  - `GET /api/v1/qrcodes/:id/scans?limit=100`
- Clés API dans dashboard (avec rotation)
- Rate limits par tier (100/h Pro, 1000/h Business)
- Documentation OpenAPI / Scalar UI
- SDK JavaScript/TypeScript

### 3.3 Webhooks
- Events : `qr.scanned`, `qr.created`, `qr.expired`, `subscription.updated`
- Signature HMAC-SHA256 (comme Stripe)
- Retry automatique avec exponential backoff
- Dashboard webhook : logs + replay

### 3.4 Domaines personnalisés
- Business tier : CNAME sur son domaine
- Au lieu de `qr-aft.vercel.app/r/token` → `qr.monsite.com/produit-x`
- Configuration DNS assistée
- Certificats SSL auto (Vercel)

### 3.5 Protection par mot de passe
- Pro : ajouter mot de passe sur QR
- Page intermédiaire avant redirection
- Useful pour QR privés (docs internes, tarifs B2B)

### 3.6 A/B testing
- Associer 2+ URLs à un QR
- Répartition configurable (50/50, 70/30, etc.)
- Analytics comparatives par variant
- Déclaration auto du gagnant après N scans

### 3.7 White-label (Business)
- Custom branding sur pages de destination
- Logo + couleurs dans emails transactionnels
- Suppression mention "Powered by QRaft"
- Mode reseller (créer des sous-comptes clients)

## Notes stratégiques

### Ce qu'il faut absolument éviter
- **Feature creep** : ne pas tout faire en même temps
- **Over-engineering** : Redis, WebSockets, microservices = plus tard
- **Design générique** : chaque écran doit avoir une signature visuelle
- **i18n bâclée** : chaque feature ajoutée = 12 fichiers JSON à mettre à jour

### Ce qu'il faut viser absolument
- **Wow-effect** à chaque interaction (anim, détail, surprise)
- **Time-to-value** < 60 secondes pour créer un premier QR
- **Mobile-first** : 70% du trafic QR vient de mobile
- **Data-driven** : PostHog sur tout, décisions basées sur usage réel
- **Vitesse** : LCP < 1.5s, INP < 100ms sur toutes les pages

### Stack additionnelle à introduire
- **TanStack Query** (fetching/cache)
- **Zustand** (état global si nécessaire)
- **Framer Motion** (animations)
- **Radix UI** (primitives accessibles)
- **Vitest + Playwright** (tests)
- **Sentry** (monitoring)
- **Upstash Redis** (cache + rate limit)
- **Resend** (emails transactionnels)
- **Contentlayer** ou **fumadocs** (blog MDX)

### Métriques de succès à tracker
- **Activation** : % users qui créent leur premier QR
- **Conversion Free → Pro** : objectif 3% mois 1, 5% mois 6
- **Churn Pro** : < 5% mensuel
- **MRR croissance** : +20% MoM
- **LTV / CAC** : > 3x
- **NPS** : > 50
